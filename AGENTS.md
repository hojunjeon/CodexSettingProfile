# CodexSettingProfile 작업 규칙

이 저장소는 Codex 작업 환경을 재현 가능한 프로필 단위로 보관하는 저장소입니다. 목표는 새 PC나 새 워크스페이스에서도 저장소 안의 문서와 스크립트만 보고 같은 세팅을 만들 수 있게 하는 것입니다.

## 프로필 작성 원칙

- 새 프로필은 `profileN/` 폴더 하나로 완결되게 만듭니다.
- 각 프로필에는 최소한 `README.md`, 설치 스크립트, 검증 스크립트, 필요한 플러그인/패치/예시 config를 함께 둡니다.
- 설치 절차는 대화 문맥이나 작성자 기억에 의존하지 않아야 합니다.
- 가능한 입력값은 버전이나 commit hash로 고정합니다.
- 설치 스크립트는 재실행 가능해야 하고, 기존 사용자 파일을 바꾸기 전에는 백업해야 합니다.
- 검증 스크립트는 실패를 경고로만 넘기지 말고 실패 exit code를 반환해야 합니다.

## 금지 항목

- API 키, GitHub 토큰, 비밀번호, keyring, credential cache를 커밋하지 않습니다.
- Codex Desktop 런타임 절대경로, hook trust hash, 로컬 세션 기록처럼 머신별 상태를 프로필의 필수값으로 만들지 않습니다.
- `@latest`처럼 시간이 지나면 결과가 바뀌는 설치 지시를 기본값으로 두지 않습니다.
- 검증되지 않은 벤치마크 수치를 성공 근거처럼 쓰지 않습니다.

## 문서 기준

- 루트 `README.md`는 저장소 목적과 프로필 목록만 짧게 설명합니다.
- 각 `profileN/README.md`는 요구사항, 설치 명령, 설치되는 항목, 고정 버전, 검증 기준을 포함합니다.
- 사용자가 파일만 읽고 실행할 수 있어야 하므로, 수동 단계가 있으면 명령과 판단 기준을 함께 적습니다.
- 한국어 사용자를 기준으로 문서를 작성하되, 명령어와 파일명은 원문 그대로 둡니다.
